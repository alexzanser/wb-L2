Что выведет программа? Объяснить вывод программы. Объяснить внутреннее устройство интерфейсов и их отличие от пустых интерфейсов.

```go
package main

import (
	"fmt"
	"os"
)

func Foo() error {
	var err *os.PathError = nil
	return err
}

func main() {
	err := Foo()
	fmt.Println(err)
	fmt.Println(err == nil)
}
```

Ответ:
```
Есть два типа структур, определяющих интерфейсы в go: eface и iface.

eface - представляет интерфейс, для которого не определены методы (interface{}).
type eface struct {
	_type *_type
	data  unsafe.Pointer
}

У этой структуры два поля :
 data - указатель на реальный объект,
 _type - структура, определяющая тип объекта, реализующего интерфейс.

#---------------------------------------------------------------------------------------
iface - представляет интерфейс, для которого определены методы. 

type iface struct {
	tab  *itab
	data unsafe.Pointer
}

data - указатель на конкретный объект.
tab - структура, содержащяя дополнительные сведения об инерфейсе и типе, который его реализует.

type itab struct {
	inter *interfacetype
	_type *_type
	hash  uint32
	_     [4]byte
	fun   [1]uintptr
}

interfacetype - описывает тип самого интерфеса
_type - тип объекта,  реализующего интерфейс.
fun - указатель на список методов.


В приведенном коде интерфейс error реализует конкретный тип os.PathError, в таком случае, значение переменной err не будет равно nil, поскольку переменная типа интерфейс равна nil только тогда когда оба ее поля равны nil, однако, поле tab не пустое и несет информацию о типе os.PathError.
```
